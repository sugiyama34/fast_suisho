# fast_suisho

Suisho 11 NNUE 評価関数の高速化プロジェクト。計画は `docs/PLAN.md`、
手法サーベイは `docs/survey/`、各実験は `experiments/NNN-name/` を参照。

## 言語・表記規約

- リポジトリ内のドキュメント (`docs/`, `experiments/` のレポート等) は**日本語**で書く
- Claude とのチャット (会話) は **English**
- 将棋・将棋AI 固有の用語は日本語のまま使う (例: 評価関数, 定跡, 互角局面集, 枝刈り,
  探索部, 教師局面, 自己対局)。無理に英訳しない

## GitHub の TeX (math) レンダリング制約

GitHub は markdown のエスケープ処理を math より先に行うため、記号 (`!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~`)
に対するバックスラッシュ命令は math に届く前に壊れる。docs 内の数式では:

- `\{` `\}` は使わない → `\lbrace` `\rbrace` を使う (英字コマンドは無事に届く)
- `\,` `\!` `\;` などの記号スペーシング命令は使わない → 普通のスペースか `\ ` (backslash+space) で代用
- `\_` を math 内で使わない → 識別子 (`FV_SCALE` 等) は math の外に出しインラインコードで書く
- インライン `$...$` の開始 `$` の直前に全角文字 (`。` `、` など) を置かない → 半角スペースを挟む
- 数字の桁区切り `{,}` (バックスラッシュなしの brace) は問題なし

# fast_suisho

Suisho 11 NNUE 評価関数の高速化プロジェクト。計画は `docs/PLAN.md`、
手法サーベイは `docs/survey/`、各実験は `experiments/NNN-name/` を参照。

## 言語・表記規約

- リポジトリ内のドキュメント (`docs/`, `experiments/` のレポート等) は**日本語**で書く
- Claude とのチャット (会話) は **English**
- 将棋・将棋AI 固有の用語は日本語のまま使う (例: 評価関数, 定跡, 互角局面集, 枝刈り,
  探索部, 教師局面, 自己対局)。無理に英訳しない
- 用語の選択: 「**マス**」を使う (「升」は使わない)。「**相手**」を使う (「敵」は使わない —
  例: 相手玉, 相手側)

## GitHub の TeX (math) レンダリング制約

GitHub の math には処理経路が 2 つあり、制約が異なる (PR #6 で実測):

- **ブロック形式** (`$$` をそれぞれ単独行に置いた複数行ブロック、または ` ```math ` フェンス):
  中身は markdown のエスケープ処理を受けない。`\\`, `\{`, `\begin{cases}` / `\begin{align*}`
  等のフル LaTeX が使える。**表示数式 (display math) は必ずこの形式で書く**
- **インライン形式** (`$...$`、および 1 物理行に収めた `$$...$$`):
  markdown のエスケープ処理が先に走り、記号へのバックスラッシュ命令
  (`\{` `\,` `\!` `\;` `\\` `\_` 等) は math に届く前に壊れる。制約:
  - `\{` `\}` → `\lbrace` `\rbrace` を使う (英字コマンドは無事に届く)
  - `\,` `\!` `\;` `\\` は使わない → スペースか `\ ` (backslash+space) で代用
  - `\_` は使わない → 識別子 (`FV_SCALE` 等) は math の外に出しインラインコードで書く
  - 開始 `$` の直前に全角文字 (`。` `、` など) を置かない → 半角スペースを挟む
  - 数字の桁区切り `{,}` (バックスラッシュなしの brace) は問題なし
